DESCRIPTION
===========================================================
A BepInEx mod for Casualties: Unknown Demo that brings 16 Tarkov-style combat stimulants, 12 medical items, and a complete medical system into the game.

Features:
- 16 custom stimulant injectors with unique effects and side effects (no temporary skill level boosts)
- 12 medical items (first aid kits, surgery kits, ointments, pills)
- Starting item grants + world loot spawning
- Intelligence-based Recognition system (hide names/effects when INT too low)
- Buff indicator via native MoodleManager
- Tunnel vision URP Vignette post-processing
- Multi-language support (EN/zh_CN)
- CUCoreLib hard dependency (non-destructive save coordination)
- Medical effect serialization system (Eff.Ser/Res) — saves/restores all active stimulant effects across floors
- KrokMP multiplayer full support:
  - Per-player effect controllers run independently on each client
  - Health sync interception prevents host from overriding client stimulant effects
  - Client inventory save/restore on reconnect (items + medical effects)
  - Buff indicator correctly shows only local player's effects
- Console spawn support for all custom items
- Hover description (SHIFT to expand effect details)

Stimulant Overview (v0.2.9 balance changes):
- SJ1: Changed from STR+5/RES+3 → stamina cap +10%, stamina recovery +50% (+ mild opiate)
- SJ12: Removed temporary RES+2 toughness boost
- PNB: Removed temporary RES+3 resilience boost (keeps nail repair + muscle healing + permanent STR-1 penalty)
- Obdolbos2 permanent skill boosts remain unchanged
- All temporary skill level increases removed — only permanent changes remain (Propital/SJ9 penalties, Obdolbos/Obdolbos2 permanent boosts)

===========================================================
REQUIREMENTS
===========================================================
- BepInEx 5.x
- Casualties: Unknown Demo
- CUCoreLib (required — hard dependency for save coordination and item registration)
- Optional: QoL Unknown (save compatibility handled automatically via CUCoreLib)
- Optional: KrokMP (multiplayer safe mode auto-detected)

===========================================================
INSTALLATION
===========================================================
1. Download CUTarkovMedicalMod_v0.2.9.zip
2. Extract the contents directly into BepInEx/plugins/CUTarkovMedicalMod/
   (DLL, Framework/, and Lang/ should be at the top level)
3. Ensure CUCoreLib.dll is in BepInEx/plugins/
4. Launch the game

v0.2.7 - 2026-07-14
- Fix: Injectors stored inside containers (medkit) reverted to base "syringe" when
  transitioning to the next floor
  - Root cause: QoLSaveFix Phase 2 only searched by custom ID, but container items
    still had base prefab ID ("syringe") after Prefix replacement. Phase 1 only
    converted items directly in body slots, not container children.
  - Fix: Phase 2 now falls back to searching by base prefab ID when custom ID search
    fails, then converts the found item back to custom. Uses HashSet to avoid
    duplicate matches when multiple injectors share the same base prefab.
- Fix: Custom injectors disappeared when held in main hand with CUCoreLib installed
  - Root cause: CUCoreLib ApplyCustomItemComponents recalculates condition as
    liquidAmount/capacity. With capacity=1 and no liquid contents, condition=0,
    causing the item to vanish even with destroyAtZeroCondition=false.
  - Fix: Set capacity=0 (skips condition recalculation) + defaultContents=[{water,0}]
    (triggers "waterbottle" template in ChooseTemplateId, avoiding "bandage" which
    caused IndexOutOfRangeException). condition stays at 1.0 as set by ConfigureSpawnedItem.
- Refactor: Centralized item definitions in MedicalItemDefinitions (shared by both modes)
